# 超售的数学原理

## 问题抽象

先把题目里的口径统一成一组变量：

- 设 `T` 为 1 个理论商品的理论单价（元）。
- 设第 `i` 个用户每月真实消耗为 `X_i`，单位是”理论商品个数”（个）。
- 用户消耗区间满足 `0 <= X_i <= 3`（个）。
- 轻度用户大致落在 `X_i <= 0.3`（个），并且已知至少 `70%` 的用户满足这个条件。
- 最保守地看，剩余 `30%` 用户都按重度用户 `X_i = 3`（个）计算。
- 管理员采购 1 个”实际商品”的成本是 `0.5T`（元），但它可以承载 `3T`（元）的理论价值。

这意味着：

- 1 个理论消耗单位对应的实际成本（元/个）是

```math
c = \frac{0.5T \text{（元）}}{3 \text{（个）}} = \frac{T}{6} \text{（元/个）}
```

- 如果一共有 `N` 个用户，则总消耗（个）为

```math
S_N = \sum_{i=1}^{N} X_i \text{（个）}
```

- 人均消耗（个/人）为

```math
\bar X_N = \frac{S_N \text{（个）}}{N \text{（人）}} \text{（个/人）}
```

## 用户数与实际单价的函数关系

如果管理员希望维持 `20%` 的盈利率，并按常见的“成本加成”口径理解为：

```math
\text{售价} = 1.2 \times \text{成本}
```

那么 `N` 个用户池对应的保本实际单价（元/人）函数就是：

```math
p_{\min}(N) = 1.2 \cdot c \text{（元/个）} \cdot \bar X_N \text{（个/人）} = 1.2 \cdot \frac{T}{6} \cdot \bar X_N = 0.2T \cdot \bar X_N \text{（元/人）}
```

这条式子的含义很直接：

- 人均消耗 `\bar X_N` 越高，至少应该收取的实际单价越高。
- `N` 本身并不会直接改变均值，但会显著降低均值的波动。
- 所以用户数越多，超售模型越稳定，越接近“按平均数赚钱”而不是“被个别重度用户击穿”。

## 在题目约束下的保守均值上界

由于只知道“70% 用户消耗不超过 0.3”，不知道更细的分布，所以用最保守的两点分布估计：

```math
X =
\begin{cases}
0.3, & 70\% \\
3, & 30\%
\end{cases}
```

则人均消耗上界（个/人）为：

```math
\mu_{\max} = 0.7 \times 0.3 + 0.3 \times 3 = 1.11
```

代回上式，可得期望意义下的最低可持续售价（元/人）：

```math
\mathbb E[p_{\min}] \le 0.2T \times 1.11 = 0.222T
```

题目里给出的实际单价是 `0.5T`，明显高于 `0.222T`。  
这说明在“70% 轻度用户、30% 全按重度用户处理”的极保守假设下，超售在均值层面是成立的。

## 20% 盈利目标对应的最大可承受人均消耗

题目已经把实际售价固定为 `0.5T`。  
若要维持 `20%` 的成本加成盈利率，则可承受的最大人均消耗 `x_*`（个/人）满足：

```math
0.5T = 0.2T \cdot x_*
```

所以：

```math
x_* = 2.5
```

也就是说，只要用户池的人均真实消耗不超过 `2.5` 个/人/月，这个定价就还能维持 `20%` 盈利率。

而前面算出的保守均值上界只有 `1.11`，安全垫非常厚：

```math
2.5 - 1.11 = 1.39
```

## 如何把“用户数”纳入模型

真正决定”至少要招揽多少用户”的，不是均值，而是波动。

前面算出的 `μ = 1.11` 是**期望**均值，但现实中 `N` 个用户的实际人均消耗 `\bar X_N` 会随机波动，可能高于 `μ`。用户数少时波动大，可能被几个重度用户拉爆；用户数多时波动小，实际值会收敛到期望附近。

要量化这个风险，需要回答一个问题：**实际人均消耗偏离期望超过多少的概率有多大？**

这正是 Hoeffding 不等式擅长的场景。它的适用条件是：

1. 各 `X_i` 相互独立（不同用户的消耗互不影响）——合理假设。
2. `X_i` 有界，即 `X_i ∈ [a, b]`——本题中 `X_i ∈ [0, 3]`，满足。

在此条件下，Hoeffding 不等式给出：

```math
\Pr(\bar X_N - \mu \ge t)
\le
\exp\left(-\frac{2Nt^2}{(b-a)^2}\right)
```

各符号含义：

| 符号 | 含义 | 本题取值 |
|------|------|----------|
| `\bar X_N` | N 个用户的**实际**人均消耗（个/人），是随机变量 | — |
| `\mu` | 人均消耗的**期望**值（个/人） | `1.11` |
| `t` | 实际人均消耗超过期望的幅度（个/人），`t > 0` | 待定 |
| `N` | 用户数（人） | 待求解 |
| `b - a` | 单个用户消耗 `X_i` 的取值范围宽度（个），即 `b - a = 3 - 0 = 3` | `3` |
| `\Pr(\cdot)` | 概率 | — |
| `\exp(\cdot)` | 自然指数函数 `e^{(\cdot)}` | — |

不等式读作：**实际人均消耗比期望高出至少 `t` 的概率，不超过右边那个指数上界。** 右边随 `N` 增大指数衰减——用户越多，大幅偏离的概率越低。

为什么选 Hoeffding 而不是中心极限定理（CLT）？因为 CLT 要求 `N` 较大才近似成立，而本题的用户数可能只有个位数，Hoeffding 对任意 `N` 都成立且不依赖分布形状，只是给出的界偏松（保守），适合做”最坏情况估算”。

取最保守均值 `\mu = 1.11`，并要求不亏于 20% 盈利目标，也就是：

```math
\bar X_N \le 2.5
```

此时

```math
t = 2.5 - 1.11 = 1.39
```

于是有：

```math
\Pr(\bar X_N > 2.5)
\le
\exp\left(-\frac{2N \times 1.39^2}{9}\right)
\approx
\exp(-0.429N)
```

这条式子可以直接反推出“至少需要多少用户”。

## 最小用户数结论

### 按 95% 把握来做

要求亏损概率不超过 `5%`：

```math
\exp(-0.429N) \le 0.05
```

解得：

```math
N \ge 7
```

### 按 99% 把握来做

要求亏损概率不超过 `1%`：

```math
\exp(-0.429N) \le 0.01
```

解得：

```math
N \ge 11
```

## 更保守的口径

如果把“盈利率 20%”理解成更严格的“净利率 = 利润 / 收入 = 20%”，那么允许的最大人均消耗会从 `2.5` 降到：

```math
x_* = \frac{0.5T \times (1-0.2)}{T/6} = 2.4
```

此时用同样方法计算：

- `95%` 把握下，至少需要 `9` 个用户。
- `99%` 把握下，至少需要 `13` 个用户。

## 建议

如果只是做小规模试水：

- 至少应有 `7` 个用户，这样在成本加成口径下有约 `95%` 的把握维持目标盈利。

如果要把这个价格当成正式可长期执行的超售策略：

- 建议按更保守的净利率口径做约束，至少招揽 `13` 个用户再开池。
- 实务上建议把启动门槛设为 `15` 个用户，会比 `11` 或 `13` 个更稳，能更好吸收偶发的重度用户聚集。

## 一句话结论

在题目给定的分布假设下，`0.5T` 的实际单价是足够赚钱的，问题不在“均值够不够”，而在“用户池够不够大”。  
如果想把超售做成稳定策略，而不是赌运气，建议至少招揽 `13` 个用户，最好 `15` 个用户以上再开始正式售卖。
