# v1.2.17 CI 测试与 Lint 失败修复计划

## 背景

v1.2.17 引入插件系统（PluginService）后，CI 工作流中 golangci-lint 和 test 两个 Job 失败。Release 构建不受影响（已成功发布 Docker 镜像），但这些测试债务需要清理。

## 问题清单

### golangci-lint 失败（3 个错误）

| # | 文件 | 错误 | 原因 |
|---|------|------|------|
| 1 | `internal/service/dashboard_oversell_service_test.go:10` | `undefined: newMockSettingRepo` | 插件系统重构了 SettingRepo 的 mock，旧的 `newMockSettingRepo` 函数被删除或重命名，但此测试文件仍引用旧名 |
| 2 | `internal/service/dashboard_oversell_service_test.go:30` | `undefined: newMockSettingRepo` | 同上，同一函数的第二次调用 |
| 3 | `internal/service/openai_gateway_record_usage_test.go:260` | `undefined: ptr` | 引用了未导入的 `ptr` 辅助函数（可能是自定义的指针工具函数或被移除的依赖） |

### test 失败（3 个 API Contract 用例）

| # | 用例 | 差异 | 原因 |
|---|------|------|------|
| 1 | `POST /api/v1/keys` | expected `len=23` → actual `len=24`，多了 `plugin_settings: map[string]interface{}{}` | 插件系统在 API Key 响应中新增了 `plugin_settings` 字段，但 `api_contract_test.go` 的期望 fixture 没有同步更新 |
| 2 | `GET /api/v1/keys (paginated)` | expected `len=23` → actual `len=24`，多了 `plugin_settings` | 同上 |
| 3 | `GET /api/v1/admin/settings` | expected `len=113` → actual `len=114`，多了 `enable_anthropic_cache_ttl_1h_injection: false` | Settings 响应新增了 `enable_anthropic_cache_ttl_1h_injection` 字段，fixture 未同步 |

## 修复方案

### 1. 修复 `dashboard_oversell_service_test.go`

- 定位 `newMockSettingRepo` 的新名称或替代方案
- 更新第 10 行和第 30 行的调用，使其引用正确的 mock 构造函数

### 2. 修复 `openai_gateway_record_usage_test.go`

- 确认 `ptr` 函数的来源（项目内自定义工具函数 or 第三方包）
- 添加缺失的 import 或替换为项目内现有的指针工具函数

### 3. 修复 `api_contract_test.go`（3 个用例）

- 在 `POST /api/v1/keys` 的期望响应中添加 `"plugin_settings": map[string]interface{}{}`
- 在 `GET /api/v1/keys (paginated)` 的期望响应中添加 `"plugin_settings": map[string]interface{}{}`
- 在 `GET /api/v1/admin/settings` 的期望响应中添加 `"enable_anthropic_cache_ttl_1h_injection": false`

## 验证

修复后本地运行：

```bash
cd backend
golangci-lint run ./...
go test -tags=unit ./...
```

确认两个命令均通过后再提交。
