# AGENTS.md 优化计划

## 背景

AGENTS.md 作为"跨平台通用项目指令（Single Source of Truth）"，经长期有机生长，已出现以下问题：

- 与 CLAUDE.md 存在大量内容重复（DRY 违反）
- 文档自身内冗余（同一观点多处表述）
- 平台特定内容混入通用层
- 部分章节对 AI 执行而言信息密度偏低

## 优化目标

1. **消除重复**：AGENTS.md 是 Single Source of Truth，CLAUDE.md 只保留 Claude Code 特有内容
2. **提高信息密度**：删除对 AI 执行无指导意义的说明性文字
3. **改善结构**：按"身份 → 环境 → 规则 → 流程 → 边界"的逻辑重新组织
4. **平台内容分离**：Codex CLI 说明独立为 CODEX.md

---

## 诊断：具体问题与优化措施

### P1. AGENTS.md ↔ CLAUDE.md 重复（高优先级）

CLAUDE.md 通过 `@./AGENTS.md` 自动加载 AGENTS.md 全部内容，但 CLAUDE.md 自身又重复抄写了以下内容：

| CLAUDE.md 中的重复章节 | AGENTS.md 中的对应章节 |
|---|---|
| 后端（Go）开发规范 | 关键注意事项 → Ent Schema 变更流程 + Interface 变更 |
| 前端（Vue3）开发规范 | 技术栈 + 核心工作流 → 前端开发 |
| README 文档 | README 维护规则 |
| docs 文档 | docs 非计划类文档同步 |
| 远程测试站点 | 远程测试站点约定 |
| 测试 | 开发命令 → 后端/前端 |

**措施**：精简 CLAUDE.md，只保留 Claude Code 特有的操作指令（文件引用语法、TodoWrite 用法、Edit 工具偏好），删除所有与 AGENTS.md 重复的规范内容。

### P2. AGENTS.md 内部冗余（中优先级）

| 冗余点 | 出现位置 | 措施 |
|---|---|---|
| "不使用序号前缀" | KISS 原则 + 奥卡姆剃刀原则 + 有机更新原则（3 处） | 只在 KISS 中保留一次 |
| Ent Schema 变更流程 | "关键注意事项" + "开发命令 → 后端" | 合并到"开发命令"，注意事项中只保留**必须**级别的提醒 |
| pnpm-lock.yaml 同步 | "关键注意事项" + "前端开发工作流" + 技术栈表格 | 合并到前端开发工作流 |
| docs 文档同步规则 | "关键注意事项" 全节 | 精简为一句话规则，去掉解释性段落 |
| 远程测试站点 | 5 条 bullet，含大量解释 | 精简为 3 条核心规则 |

### P3. Codex CLI 内容分离（中优先级）

AGENTS.md 中的"Codex CLI 特定说明"章节（约 25 行）是平台特定内容，应与 Claude Code 的 CLAUDE.md 同级处理：

**措施**：
- 新建根目录 `CODEX.md`，内容以 `@./AGENTS.md` 引用通用指令 + Codex CLI 特定说明
- 从 AGENTS.md 中移除"Codex CLI 特定说明"整节
- 理由：AGENTS.md 定位为"跨平台通用"，不应包含任何单一平台的操作指令

### P4. 章节精简（低优先级）

| 章节 | 当前状态 | 优化方向 |
|---|---|---|
| 版本号管理规范 | 含语义化版本教科书式说明 | 精简为版本号规则表 + 更新时机，删除对 semver 规范的科普 |
| 变更记录规范 | 含完整 Keep a Changelog 模板 | 精简为格式示例 + 记录范围 + 记录时机，删除对 KaC 规范的科普 |
| 有机更新原则 | 含"理解意图/定位生态位/协调生长/保持呼吸感/定期修剪"5 个子节 | 精简为 2-3 条核心规则（同步更新、格式规范、臃肿时重构） |
| 工程原则表 | 8 行表格 | 合并 KISS 和奥卡姆剃刀（核心思想高度重叠），精简"在本项目中的体现"列 |
| README 维护规则 | 7 条 bullet | 合并为 4 条核心规则 |

## 执行步骤

### 步骤 1：创建 CODEX.md

从 AGENTS.md 提取"Codex CLI 特定说明"章节，包装为独立平台配置文件。

### 步骤 2：重写 AGENTS.md

按目标结构重写，执行 P1-P4 所有优化措施。

### 步骤 3：精简 CLAUDE.md

删除所有与 AGENTS.md 重复的规范内容，只保留 Claude Code 特有指令。

### 步骤 4：验证一致性

- 确认三个文件之间无遗漏、无矛盾
- 确认 CLAUDE.md 的 `@./AGENTS.md` 引用正常
- 确认 AGENTS.md 内无重复表述

### 步骤 5：更新 CHANGELOG.md

在 `[Unreleased]` 下记录本次优化。

## 风险与注意事项

- **不改变语义**：所有优化仅改变表述形式和结构，不改变任何规则的含义或约束力
- **不删除规则**：每条现有规则在优化后都有对应的新位置
- **向后兼容**：CODEX.md 新建不影响现有工作流；AGENTS.md 的 URL 和锚点可能有变化但影响极小
