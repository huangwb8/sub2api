# sub2api-add-users

基于真实运营数据，评估某个 sub2api 站点当前适合新增多少用户。

## 触发场景

当你想判断一个真实运营中的 sub2api 站点：
- 现在有没有算力冗余
- 最多还能新增多少同类用户
- 是否已经存在用户容量缺口

就用这个 skill。

## 输入

- 站点基础地址
- 管理员只读鉴权信息
- 可选：时间段，默认最近 7d
- 可选：时区，默认 `Asia/Shanghai`

## 输出

在 `./tmp/sub2api-add-users/run-{时间戳}/report.md` 写出最终结论：
- 正值：还能新增这么多用户
- 0：先观察
- 负值：当前已经有容量缺口

## 工作流

1. 只读采集必要的运营数据
2. 结合源码核对容量推荐口径
3. 写出分析与报告

## WHICHMODEL

- 首选：`gpt-5.5`，适合首轮分析、跨文件核对和容量口径推理。
- 轻量修改：`gpt-5.4`，适合仅调整文案、路径、模板或小范围脚本。
